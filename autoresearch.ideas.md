- Highest-priority next step: replace the autoregressive stage-1 Q0 LM with a non-autoregressive or semi-parallel text-to-frame semantic predictor. Encode text once with the pretrained Qwen prior, expand it onto a frame axis, and predict all Q0 tokens (or frame blocks) together. The repeated pattern is unchanged: teacher-forced learning improves, but free-running autoregressive Q0 decoding collapses.
- A learned duration / frame-count predictor is now a secondary clue, not a mainline answer. Several runs showed it can replace the hard 121-frame cap with much more realistic lengths (roughly 32–69 frames OOD) and sometimes improve CER, but by itself it still has not improved OOD `asr_wer_prefix`. If revisited, it should probably be paired with a fundamentally stronger semantic decoder rather than treated as the main fix.
- If revisiting exposure-bias work, prefer more structural generated-prefix training over scalar retuning: e.g. train stage 1 on longer model-generated rollouts, grouped/blockwise semantic decoding, or another objective that directly matches free-running inference. The current scheduled-sampling win (`semantic_self_condition_prob=0.5`, `semantic_eos_weight=2.0`) still looks fragile.
- Treat explicit text-conditioning tricks as mostly exhausted for now. Mixed normalized+raw text, text-summary adapters, text-token interpolation adapters, text-hidden-state additive context, a shared cross-attention decoder path, and duration-cap combinations all improved at most secondary signals without improving the OOD primary metric.
- Lowering `length_percentile` to focus on shorter utterances also looks stale: it reduced the retained train set and shortened the cap, but did not improve WER.
