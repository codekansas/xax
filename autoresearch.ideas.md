- Current best recipe still looks like a **narrow real sweet spot**: `semantic_future_prediction_steps=4`, `semantic_future_prediction_weight=0.5`, `semantic_self_condition_prob=0.5`, `semantic_eos_weight=2.0`, and the p95-style 53-frame cap. It reproduced `asr_wer_prefix=0.6667` twice. Nearby scalar retunes remain stale: future-loss weight `0.25` or `1.0`, five future steps, looser/shorter caps, frozen-Qwen / `use_lora=false`, stronger Stage-1 loss weighting, and semantic/residual LR retunes all regressed.
- **Scheduled sampling is still essential even in the stronger structural regime.** Ablating `semantic_self_condition_prob=0.5` from the four-head future-Q0 winner sent `asr_wer_prefix` back to `1.0`, despite much faster throughput. So the multi-token generated-prefix auxiliary does not replace exposure-bias training by itself.
- **Train/infer-matched grouped blockwise semantic decoding is still the strongest remaining branch, but its local neighborhood is now tighter.** The best version remains: `text_source=both`, `semantic_future_prediction_steps=2`, `semantic_future_prediction_weight=0.5`, `semantic_self_condition_prob=0.5`, three-token blockwise semantic decoding, and the same 53-frame cap. That branch tied the current best at `asr_wer_prefix=0.6667`.
- The grouped/blockwise branch now has several more stale local retunes: increasing the auxiliary weight to `0.6` or `0.75` regressed, shortening the cap to 48 frames regressed, warmup schedules before grouped decoding regressed, and lowering scheduled sampling to `semantic_self_condition_prob=0.25` also regressed. So if grouped decoding is revisited, keep **self-conditioning at 0.5**, **future weight at 0.5**, and the **53-frame cap** fixed unless there is a more structural reason to change them.
- The broader blockwise picture still suggests a **small sweet spot**. Naive block size `2` helped CER but not WER; naive size `3` reached `0.8333`; naive size `4` regressed badly. Matched `1 head -> 2-token blocks` reached `0.8333` with very strong in-domain WER when `text_source=both`, but the stronger matched `2 heads -> 3-token blocks` is the only grouped path that can tie the best primary metric.
- **Simple residual-robustness noise now also looks stale in the stronger structural regime.** Adding mild `q0_corruption_prob=0.05` and `residual_semantic_hidden_dropout_prob=0.05` on top of the four-head winner regressed back to `asr_wer_prefix=1.0`, so orthogonal stage-2 noise is not an easy win.
- That leaves two active directions:
  - keep pushing **more structural grouped/blockwise train/infer match** ideas, not scalar tweaks — for example a new grouped decoding mechanism or training target that better matches how grouped tokens are consumed at inference;
  - if that stalls, pivot to a **stronger semi-parallel / hierarchical Stage-1 decoder** rather than the old lightweight non-AR interpolator.
- The simple non-autoregressive text-to-frame Q0 upsampler still looks underpowered in its current form. Both the interpolation-only and packed-prefix variants trained fast and controlled length, but OOD `asr_wer_prefix` never improved. If revisited, it should be in a stronger form (hierarchical/blockwise/alignment-aware), not just a larger version of the same lightweight upsampler.
- Treat explicit text-conditioning tricks and duration-only fixes as exhausted supporting ideas: mixed normalized+raw text helps some grouped variants, but adapters / cross-attention / interpolation context / duration heads / prompt prefixes have not directly improved the primary OOD metric.
