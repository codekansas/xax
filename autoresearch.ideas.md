- Replace the autoregressive stage-1 Q0 LM with a non-autoregressive text-to-frame semantic upsampler: encode text once with frozen Qwen, interpolate/cross-attend those hidden states onto a frame axis, and predict all Q0 tokens in parallel. This directly attacks the current failure mode (teacher-forced loss improves but free-running collapses).
- Add an explicit duration / frame-count predictor so inference no longer depends on semantic EOS. The current model learns EOS under teacher forcing but never emits it during generation; a learned length head plus fixed-frame decoding could remove that bottleneck entirely.
- Try scheduled-sampling style stage-1 training that mixes teacher-forced Q0 histories with model-generated or corrupted prefixes more aggressively than the current token corruption knob. The gap between ~50% teacher-forced Q0 accuracy and 0% useful free-running WER suggests exposure bias is the main semantic failure.
- If keeping the autoregressive stage-1, add a small text-conditioning adapter or cross-attention block rather than relying only on LoRA over the whole decoder. Current prompt-prefix and raw-text tweaks were not enough to preserve OOD text conditioning.
